# LiteBot

A lightweight QQ bot built on [NoneBot2](https://nonebot.dev/) and the OneBot v11 protocol.

[简体中文](README.md)

> [!WARNING]
> Almost all code in this repo is generated by AI agents and may contain bugs, security issues, or just plain nonsense.
> Though I manually review and test all code before merging, I can't guarantee its correctness or safety.
> Use at your own risk and always review the code before running it.

## Features

| Plugin | Command(s) | Description |
|---|---|---|
| **service** | `/svc` | Per-group / per-user feature toggle manager |
| **ping** | `/ping` | Bot status and event-delivery latency check |
| **help** | `/help` | Browse loaded plugin documentation |
| **mute** | — | Check if bot is silenced in the current context to avoid resource-wasting operations |
| **withdraw** | `/withdraw` | Recall recently sent bot messages |
| **login_notice** | — | Notify superusers when the bot logs in |
| **group_notice** | `/notice` | Configurable group join / leave messages |
| **anti_miniapp** | — | Extract urls from miniapp messages |
| **exception_report** | — | Forward unhandled exceptions to configured superusers |
| **ntfy** | — | Bridge [ntfy.sh](https://ntfy.sh) push notifications to QQ groups/users |
| **drasl** | `/invite` | Generate [Drasl](https://github.com/unmojang/drasl) Minecraft auth-server invite links |
| **b23extract** | — | Parse Bilibili links and reply with rich info cards |
| **mcping** | `/mcping` | Ping a Minecraft server and show player/version info |

## Requirements

- Python ≥ 3.14
- [uv](https://github.com/astral-sh/uv) (recommended) or any PEP 517-compatible installer
- A running OneBot v11 implementation

## Quick Start

```bash
# 1. Clone the repo
git clone https://github.com/KoishiMoe/litebot.git
cd litebot

# 2. Install dependencies with uv
uv sync

# 3. Copy and edit the configuration
cp .env.example .env
$EDITOR .env          # set SUPERUSERS, credentials, etc.

# 4. Run
uv run python bot.py
```

Point your OneBot implementation at `http://<host>:8080` (default port; override with `PORT=` in `.env`).

## Configuration

All settings live in `.env` (copy from `.env.example`).  Key options:

| Variable | Default | Description |
|---|---|---|
| `HOST` / `PORT` | `0.0.0.0` / `8080` | HTTP server address |
| `COMMAND_START` | `["/"]` | Bot command prefix(es) |
| `SUPERUSERS` | `[]` | QQ UIDs with superuser privileges |
| `BOT_LANGUAGE` | `zh` | UI language (`en` or `zh`) |
| `DATA_DIR` | `data` | Root directory for persistent plugin data |
| `LOG_DIR` | `logs` | Directory for on-disk log files |
| `CARD_FONT` | *(auto)* | Path to a CJK-capable font for image cards |
| `BILIBILI_SESSDATA` / `BILIBILI_BILI_JCT` / `BILIBILI_BUVID3` | *(empty)* | Optional Bilibili credentials |
| `DRASL_SERVER` / `DRASL_TOKEN` | *(empty)* | Drasl server URL and API token |
| `NTFY_SERVER` / `NTFY_TOKEN` | `https://ntfy.sh` / *(empty)* | ntfy server and auth token |

See `.env.example` for the full list with inline documentation.

## Image Cards

The `b23extract` and `mcping` plugins can generate rich image cards.  Both use:

- **Shared font system** — auto-detected from common system paths (Noto CJK recommended: `fonts-noto-cjk` on Debian/Ubuntu).  Override with `CARD_FONT=` in `.env`.
- **Bundled Twemoji** — emoji in cards are rendered from bundled [Twemoji v17.0.2](https://github.com/jdecked/twemoji) PNG assets; no extra emoji font needed.

## Service Manager

The `service` plugin provides per-group and per-user feature gating for all other plugins.

```
/svc                          List feature status in current context
/svc on  <name>               Enable a feature
/svc off <name>               Disable a feature
/svc on  <name> @user         Enable/disable for a specific user

# Superuser (from private chat):
/svc on|off <name>                   Global toggle
/svc on|off <name> -g <group_id>     For a specific group
/svc on|off <name> -u <user_id>      For a user (all groups)
/svc on|off * [-g …] [-u …]          Wildcard: all features at once
```

## Custom Plugins

Place your own plugins under `custom/plugins/` (directory-based packages or single `.py` files) or list them in `.env`:

```
CUSTOM_PLUGINS=["my_pypi_plugin", "another_plugin"]
```
